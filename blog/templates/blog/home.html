Blog Backend


Backend Docs

A simple documentation to get you started with this simple RESTful Blog backend.


Making Blog Posts


To create blog posts, user has to be authenticated and must be the admin. Settings can be changed to suit applications. To create or list blog post, http requests must be made to the endpoint:

  • http://localhost:8000/api/blogposts/

To make GET, PATCH or DELETE requests on a blog post instance, the http verbs must act on the endpoint:
  • http://localhost:8000/api/blogposts/pk/

Example of this are http://localhost:8000/api/blogposts/1/, http://localhost:8000/api/blogposts/2/, etc.

Making Comments


To create comments, user does NOT have to be authenticated and must be the admin. Settings can be changed to suit applications. To create or list comments, http requests must be made to the endpoint:

  • http://localhost:8000/api/comments/

Only admins can perform RUD operations on comments. To make GET, PATCH or DELETE requests on a comment instance, the http verbs must act on the endpoint:
  • http://localhost:8000/api/comments/pk/

Example of this are http://localhost:8000/api/comments/1/, http://localhost:8000/api/comments/2/, etc.
